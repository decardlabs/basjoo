## 问题解决
- Python 3.14 与 pydantic-core 不兼容 → 升级 pydantic>=2.11 解决
- Python 3.14 与 SQLAlchemy 2.0.30 `__firstlineno__` 冲突 → 升级 sqlalchemy>=2.0.40 解决
- 缺少 greenlet → pip install greenlet
- Docker Hub 拉取超时 → 本地直接运行
- **BasjooWidget SSE 解析格式不匹配**：前端用 OpenAI 格式 `choices[0].delta.content` 解析，但 Basjoo API 返回 `{"content": "..."}`，导致 AI 助手无响应 → 改为优先解析 `parsed.content`，兼容 OpenAI 格式

## 完成：Decard-site + Basjoo 联合部署架构文档

### 架构说明
- 两个独立 Docker Compose 栈：`basjoo-network` 和 `decard-network`
- Basjoo 自带 Nginx 容器（:80/:443 → :8080/:8443），Decard-site 无 Nginx
- 生产环境需通过 Basjoo Nginx 或宿主 Nginx 代理 Decard 路由

| Basjoo Backend | :8000 (dev) | :8000 |
| Basjoo Frontend | :3000 (dev) | :3000 |
| Redis | 127.0.0.1:6379 | :6379 |
| Qdrant | 127.0.0.1:6333 | :6333 |
| Decard Backend | 127.0.0.1:8001 | :8000 |
| Decard Frontend | 127.0.0.1:3001 | :3000 |
