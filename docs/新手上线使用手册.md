## 1. 产品简介

### 1.1 什么是 Ccbot？

Ccbot 是一个基于 RAG（检索增强生成）技术的 AI 智能客服平台，帮助企业快速搭建智能问答系统。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 🤖 **AI 智能体** | 配置 AI 模型、系统提示词、对话行为 |
| 📚 **知识库管理** | 支持 URL 抓取和 Q&A 问答对导入 |
| 💬 **聊天 Widget** | 一行代码嵌入到任何网站 |
| 📊 **会话管理** | 查看对话记录，支持人工接管 |
| 🔒 **安全防护** | SSRF 防护、来源白名单、CORS 控制 |

### 1.3 系统架构

## 2. 快速开始

### 2.1 环境要求

| 组件 | 要求 |
|------|------|
| Docker | 20.10+ |
| Docker Compose | 2.0+ |
| 服务器 | Ubuntu 20.04+ / Debian 11+ |
| 内存 | 最少 2GB |
| 端口 | 80, 443, 8000, 3000 |

### 2.2 一键部署（推荐）

在服务器上执行以下命令：

```bash
curl -fsSL https://raw.githubusercontent.com/haoyiyin/ccbot/main/install-deploy.sh | sudo sh
```

### 2.3 Docker Compose 部署

#### 方式一：开发环境

```bash
docker compose --profile dev up -d
```

#### 方式二：生产环境

```bash
docker compose --profile prod up -d
```

### 2.4 本地开发部署

#### 后端服务

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python3 main.py
```

#### 前端服务

```bash
cd frontend-nextjs
npm install
npm run dev
```

#### Widget 开发

```bash
cd widget
npm install
npm run dev
```

### 2.5 访问地址

| 服务 | 地址 |
|------|------|
| 管理后台 | http://localhost:3000 |
| 后端 API | http://localhost:8000 |
| API 文档 | http://localhost:8000/docs |
| Qdrant | http://localhost:6333 |

### 2.6 初始化配置

1. 首次登录后台时，需要创建管理员账号
2. 配置 AI 模型服务商（OpenAI/Claude/Gemini 等）
3. 设置 Embedding 服务（Jina 或 SiliconFlow）
4. 开始添加知识库内容

## 3. 管理后台使用指南

### 3.1 登录与认证

1. 访问管理后台地址
2. 点击"注册"创建管理员账号
3. 使用账号密码登录

### 3.2 仪表盘

仪表盘展示系统概览：
- 对话统计数据
- 知识库覆盖情况
- 最近会话列表

### 3.3 Playground 测试

Playground 页面用于：
- 测试 AI 回复效果
- 观察检索结果
- 调整模型参数
- 验证知识库内容

### 3.4 智能体配置

在"智能体设置"中配置：

| 配置项 | 说明 |
|--------|------|
| 模型选择 | OpenAI / Claude / Gemini / Azure |
| 系统提示词 | 定义 AI 的角色和行为 |
| 温度参数 | 控制回复的创造性 |
| 最大 Token | 限制回复长度 |
| Widget 标题 | 聊天窗口显示的标题 |
| 欢迎语 | 首次打开的问候消息 |
| 主题颜色 | Widget 外观颜色 |

### 3.5 系统设置

- 语言切换（中/英文）
- 主题设置（深色/浅色/自动）
- Widget 外观微调
- 安全策略配置

## 4. 知识库配置

### 4.1 添加网站知识

#### 步骤 1：添加 URL

1. 进入"网站管理"页面
2. 点击"添加网站"
3. 输入目标网址（如：https://docs.example.com）
4. 点击"保存"

#### 步骤 2：抓取内容

1. 在网站列表中找到添加的 URL
2. 点击"抓取"按钮
3. 等待抓取完成

#### 步骤 3：配置自动抓取（可选）

1. 开启"自动抓取"开关
2. 设置抓取频率
3. 系统将按设定周期自动更新内容

#### 步骤 4：重建索引

> ⚠️ 重要：每次添加或更新内容后，需要重建索引才能生效

1. 进入"索引管理"页面
2. 点击"重建索引"按钮
3. 等待索引完成

### 4.2 添加 Q&A 知识

#### 单条添加

1. 进入"Q&A 管理"页面
2. 点击"添加问答"
3. 输入问题和答案
4. 点击"保存"

#### 批量导入

支持 JSON 格式导入：

```json
[
  {
    "question": "你们的工作时间是？",
    "answer": "我们的工作时间是周一至周五 9:00-18:00"
  },
  {
    "question": "如何联系客服？",
    "answer": "您可以通过以下方式联系我们：\n1. 电话：400-xxx-xxxx\n2. 邮箱：support@example.com"
  }
]
```

### 4.3 索引重建

索引重建会：
1. 清空现有向量数据
2. 重新处理所有知识内容
3. 生成新的向量索引

> 💡 建议：在完成所有知识录入后执行一次完整的索引重建

## 5. Widget 嵌入指南

### 5.1 获取嵌入代码

在管理后台的"系统设置"中找到 Widget 配置，获取嵌入代码。

### 5.2 基础嵌入

将以下代码添加到网页的 `<body>` 标签之前：

```html
<!-- Ccbot Widget -->
<script
  src="https://your-domain.com/sdk.js"
  data-agent-id="agt_xxxxxxxxxxxx"
  data-api-base="https://your-domain.com"
  defer
></script>
```

### 5.3 配置参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `data-agent-id` | 是 | 智能体 ID，从后台获取 |
| `data-api-base` | 否 | API 地址，默认自动检测 |
| `data-theme-color` | 否 | 主题颜色（十六进制） |
| `data-position` | 否 | 位置：`left` 或 `right` |
| `data-title` | 否 | 聊天窗口标题 |

### 5.4 完整示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>我的网站</title>
</head>
<body>
  <h1>欢迎来到我的网站</h1>

  <!-- Ccbot Widget -->
  <script
    src="https://your-domain.com/sdk.js"
    data-agent-id="agt_379d138bd604"
    data-api-base="https://your-domain.com"
    data-theme-color="#007AFF"
    data-position="right"
    data-title="智能客服"
    defer
  ></script>
</body>
</html>
```

### 5.5 域名白名单

为保障安全，需要在管理后台配置允许嵌入的域名：

1. 进入"系统设置"
2. 找到"Widget 域名白名单"
3. 添加允许的域名（如：`example.com`）

### 5.6 多语言支持

Widget 会自动检测访客语言并显示对应文案：
- 中文（默认）
- 英文
- 其他语言

## 6. 常见问题

### 6.1 部署问题

**Q: 部署后无法访问？**

A: 检查以下内容：
1. 确认 Docker 容器正在运行：`docker compose ps`
2. 检查端口占用：`netstat -tlnp | grep 80`
3. 查看日志：`docker compose logs -f backend`

**Q: 如何查看 API 文档？**

A: 访问 http://localhost:8000/docs 或 http://your-domain.com/api/docs

### 6.2 知识库问题

**Q: 添加的知识没有生效？**

A: 请确认已完成以下步骤：
1. ✅ 知识内容已保存
2. ✅ 索引已重建
3. ✅ 浏览器已刷新缓存

**Q: URL 抓取失败？**

A: 检查：
1. 目标 URL 是否可访问
2. URL 是否包含需要登录的内容
3. 是否有 robots.txt 限制

**Q: 如何提高检索准确率？**

A: 建议：
1. 添加更多高质量 Q&A 对
2. 优化系统提示词
3. 定期更新知识库内容
4. 在 Playground 中测试检索效果

### 6.3 Widget 问题

**Q: Widget 无法加载？**

A: 检查：
1. 确认 `data-agent-id` 正确
2. 检查浏览器控制台错误
3. 确认域名在白名单中
4. 检查 CORS 配置

**Q: 如何自定义 Widget 外观？**

A: 在管理后台的"系统设置"中调整：
- 主题颜色
- 标题文字
- 欢迎语
- Logo 图片

### 6.4 安全问题

**Q: 如何保护 API Key 安全？**

A: 系统采用以下安全措施：
1. API Key 使用 Fernet 加密存储
2. 生产环境启用 `REQUIRE_SECRET_KEY=true`
3. 敏感配置通过环境变量注入
4. 定期轮换密钥

**Q: 如何防止恶意访问？**

A: 系统提供多层防护：
1. 限流保护（默认 60 请求/分钟）
2. SSRF 防护（阻止内部网络访问）
3. 来源白名单（Widget 嵌入限制）
4. CORS 策略控制
